Kompetensi & Flow Belajar

Peta kemampuan dan alur belajar untuk membangun ekosistem FM & catering end-to-end.
Bagian 1
Checklist Kompetensi
Centang tiap poin setelah kamu merasa: paham konsep → bisa contoh sendiri → bisa pakai di proyek nyata.
Fondasi Web & Otomasi
Apps Script + HTML/CSS
Level 1
Memahami struktur HTML dasar (tag umum, form, table, layout sederhana).
Mampu membuat layout dengan Flexbox/Grid untuk dashboard sederhana (sidebar + konten).
Menggunakan Google Apps Script untuk baca/tulis data ke Google Sheets (CRUD).
Bisa membuat Web App Apps Script dengan doGet() & doPost().
Membuat form input WMS/PR/PO yang menyimpan data ke sheet dengan validasi dasar.
Mengatur trigger terjadwal (daily/weekly) untuk kirim email summary atau auto-update.
Data Backbone
SQL & Data Modeling
Level 1–2
Mengerti perintah SELECT, INSERT, UPDATE, DELETE di SQL.
Bisa melakukan JOIN antar tabel (item–recipe, PR–PO–Vendor).
Mampu mendesain ERD sederhana untuk WMS & Procurement (tabel, relasi, primary key).
Mengerti konsep normalisasi (menghindari data duplikat berlebihan).
Mengerti index & unique key dan kapan dipakai.
Interaktivitas UI
JavaScript Frontend
Level 2
Memahami DOM (document object model) & memanipulasi elemen (tambah row, ubah teks, dsb).
Bisa menggunakan Fetch API untuk memanggil Apps Script / API backend.
Mengerti async/await untuk operasi asinkron seperti load data tabel.
Membuat table interaktif (search, sort, pagination sederhana) dengan JS.
Aplikasi Backend
Laravel / Backend Framework
Level 2–3
Memahami konsep Routing → Controller → Model di framework (misal: Laravel).
Dapat membuat API endpoint untuk modul (stock, PR, PO, vendor, dll).
Membuat CRUD data menggunakan ORM (misal: Eloquent di Laravel).
Implementasi authentication & role-based access (site, HO, BOD).
Integrasi & Analitik
API & Data Warehouse
Level 3
Mengerti konsep REST API & format JSON.
Bisa mengkonsumsi atau menyediakan API untuk sistem lain (HRIS, accounting, client portal).
Mengerti konsep ETL (Extract–Transform–Load) ke data warehouse mini.
Mampu mendesain data mart untuk analitik (WMS, procurement, finance).
Advanced
PWA & Quality Engineering
Level 3–4
Memahami konsep PWA (installable, offline-capable, web manifest).
Mengerti dasar Git dan version control untuk kolaborasi.
Mengerti konsep testing (setidaknya manual structured test dan basic automation).
Mengerti konsep deployment & environment (dev, staging, production).
Bagian 2
Flow Belajar dari Awal sampai Akhir
Flow ini dibagi per fase, kira-kira 18 bulan, tapi bisa kamu percepat kalau intensitas belajarnya lebih tinggi.
Fase 1 – Fondasi Operasional Digital
(Bulan 1–2)
Fokus: Apps Script, HTML/CSS, SQL dasar.
  • Menguasai HTML & CSS dasar untuk layout dashboard & form.
  • Belajar Apps Script: CRUD ke Google Sheets, Web App, trigger.
  • SQL dasar: SELECT, INSERT, UPDATE, DELETE, JOIN.
  • Membuat WMS & PR/PO sederhana full di Google Sheets + Apps Script.
Fase 2 – Interaktivitas & Data Backbone
(Bulan 3–5)
Fokus: JavaScript frontend & desain data lebih matang.
  • Belajar JS DOM, event handling, Fetch API untuk tarik data dari Apps Script.
  • Bangun table interaktif: search, filter, pagination manual.
  • Rancang ERD untuk WMS + Procurement (item–recipe–stock–PR–PO–vendor).
  • Benahi master data & standardisasi kode (PR, PO, item, site).
Fase 3 – Backend Framework & API
(Bulan 6–9)
Fokus: migrasi ke backend modern (misal Laravel) + database SQL penuh.
  • Belajar dasar Laravel (routing, controller, model, migration).
  • Migrasi modul kritikal (WMS & PR/PO) dari Sheet ke database MySQL/PostgreSQL.
  • Membuat API endpoint untuk stock, PR, PO, vendor, GRN.
  • Implement login & role-based access: site, HO, BOD.
Fase 4 – Integrasi & Analitik Lanjut
(Bulan 9–12)
Fokus: integrasi sistem & mini data warehouse.
  • Desain pipeline ETL sederhana: dari DB operasional ke DB analitik.
  • Bangun data mart untuk cost control, vendor performance, dan SLA site.
  • Perkuat dashboard (Looker Studio / BI lain) dengan data dari SQL, bukan hanya Sheet.
  • Mulai rancang API integrasi ke HRIS / accounting / client portal.
Fase 5 – PWA & Quality Engineering
(Bulan 12–18)
Fokus: menjadikan sistem sebagai “produk” yang scalable & nyaman dipakai.
  • Belajar konsep PWA: installable app, offline cache, web manifest.
  • Implement Git & basic branching untuk versi dan backup.
  • Menyusun pengujian terstruktur (test case) untuk modul kritikal (WMS, PR/PO, cost).
  • Optimasi performa & stabilitas sistem untuk multi-site dan multi-user.
Ringkasan Jalan Ninja Kamu

Kamu tidak mulai dari nol. Kamu sudah kuat di logika, data, dan operasional. Flow ini menyusun urutan teknologi supaya:

  • Di awal: cepat dapat hasil (Apps Script + Sheets + HTML).
  • Di tengah: pindah ke sistem yang lebih kokoh (Laravel + SQL).
  • Di akhir: ekosistem siap jadi platform kelas korporasi.

Setiap fase, lihat kembali Checklist Kompetensi di atas. Targetnya: sebelum pindah fase, minimal 70% checklist terkait fase tersebut sudah bisa kamu centang.
